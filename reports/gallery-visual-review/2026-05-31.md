## UI states

### Connecting

[`App.tsx`](../../src/App.tsx) shows `connectingScreen` until first DB connect. Automated capture could not freeze this state (poll loop missed it); live timing is **&lt; 1 s** on this machine. Spinner + two-line copy hierarchy is clear; no flash of unstyled content observed on reload miss screenshot.

### Filters active

Rating **3**, color **Red**, sort **General Score** → header **(0 items)** with empty grid. Active rating pill uses accent fill (`#007acc` / `#eeeeee`) — see Lighthouse contrast finding. Empty state could use explicit “No images match filters” copy (currently bare grid).

### Stacks mode

Toggle on: count becomes **(27518 items (grouped))**; cards show stack badge (photo count), layered card chrome ([`GalleryGrid.module.css`](../../src/components/Gallery/GalleryGrid.module.css) `.stackLayer1/2`, `.stackBadge`), and **100%** score line. Visual stack metaphor reads well.

### Stack drill-down vs image viewer

- Single click on stack card → **stack drill-down** (`Stack #…`, sub-stacks) — [08-stack-drilldown.png](./08-stack-drilldown.png).
- Click on thumbnail image → **`ImageViewer`** side panel with metadata, EXIF, tagging progress — [08b-image-viewer.png](./08b-image-viewer.png).

### Light scheme

CDP `emulate({ colorScheme: 'light' })` applied; screenshot timed out. App is **dark-first** (`body` uses `var(--bg-dark)` in [`layout.css`](../../src/styles/layout.css)); no dedicated light theme in local tokens.

**Failed a11y audits (actionable for design):**

1. **Active rating button “All”** — contrast **3.88:1** (`#eeeeee` on `#007acc`, 11px). Needs **≥ 4.5:1** (sidebar filter module, class `_ratingButtonActive_`).
2. **Date filter placeholder** — “Filter by shot date…” **2.71:1** (`#5e5e5e` on `#191919`). Lighten placeholder or darken field bg.

Full JSON: [lighthouse/report.json](./lighthouse/report.json).

## Findings

| Area | Severity | Observation | Suggested fix |
|------|----------|-------------|---------------|
| Responsive layout | Medium | Fixed 250 px sidebar; grid stays 3-col down to 900 px | Add breakpoint: collapse sidebar to icon rail or overlay under ~1100 px |
| Card chrome | Low | Bright blue horizontal rule under thumbnail reads as debug/separator, not design token | Use `var(--border)` or subtle `var(--text-secondary)` divider in [`GalleryGrid.module.css`](../../src/components/Gallery/GalleryGrid.module.css) |
| Card metadata | Low | `.cardScore` uses hardcoded `#888` | Use `var(--text-secondary)` for theme consistency |
| Filter empty state | Low | 0 results shows empty grid only | Add centered empty-state message when `imagesCount === 0` and filters active |
| Connecting screen | Info | Too fast to screenshot; functionally fine | Optional minimum display or skeleton if CLS is a concern |
| Folder tree | Low | Footer shows only `/` at root | Expected at library root; verify expanded tree styling when folder selected |
| Disabled Back | Low | Gray disabled back at root is correct but heavy | Matches `.backButton:disabled` in [`FsGallery.module.css`](../../src/components/FsMode/FsGallery.module.css) — OK |
| Color contrast | Medium | Lighthouse: active rating pill + date placeholder | Adjust colors in sidebar filter CSS modules |
| Image viewer (dev) | Low | `/media/` NEF fetch cache error in browser-only path; thumbnail fallback works | Document Electron requirement; already noted in App error copy |
| Light mode | Info | No light theme | Out of scope unless product requests |
| SEO / robots | Info | Dev server only | N/A for desktop app |

## No issue (baseline)

- Dark theme cohesion (`--bg-dark`, `--card-bg`, gold stars `--score-gold`).
- Thumbnail grid virtualization loads 50 items; `media://` thumbnails render in Electron dev.
- Stacks toggle and grouped layout are visually distinct and understandable.
- WebSocket live indicator path works; header item count matches DB scale.
- Native `<select>` filters align with dark inputs.
- Accessibility tree: filter controls have labels (`Filter by keyword`, `Sort by`, color descriptions).
